\section{Token Ekonomisi ve Kota Sistemi}
% ============================================================

\subsection{Genel Mimari}
Token ekonomisi, \texttt{TokenBalance} modeli üzerinden yönetilmektedir. Her kullanıcının tek bir cüzdan kaydı (\texttt{token\_balance} tablosu) bulunur ve bu kayıt bakiye, harcama, günlük/haftalık kota ve aylık yenileme bilgilerini barındırır.

\subsection{Tek Seferlik 100 Token Kayıt Bonusu}
Yeni kayıt olan her kullanıcıya \textbf{bir defalık 100 token} verilmektedir. Bu bonus \textbf{otomatik olarak yenilenmez}; kullanıcının ek token elde etmesi için paket satın alması gerekmektedir.

\subsubsection{Teknik Uygulama}
Sabit değer \texttt{app.py} dosyasında tanımlıdır:

\begin{lstlisting}[language=Python, caption={Signup grant sabiti (app.py satır 465)}]
SIGNUP_GRANT_TOKENS = 100
\end{lstlisting}

Cüzdan oluşturma fonksiyonu, kullanıcının ilk erişiminde çağrılır:

\begin{lstlisting}[language=Python, caption={get\_or\_create\_token\_balance fonksiyonu}]
def get_or_create_token_balance(user: User) -> TokenBalance:
    wallet = TokenBalance.query.filter_by(user_id=user.id).first()
    if not wallet:
        wallet = TokenBalance(
            user_id=user.id,
            balance=SIGNUP_GRANT_TOKENS,
            total_spent=0
        )
        db.session.add(wallet)
        grant_tx = TokenTransaction(
            user_id=user.id,
            amount=SIGNUP_GRANT_TOKENS,
            type='signup_grant',
            description=f'Yeni kullanici hos geldin bonusu'
        )
        db.session.add(grant_tx)
        db.session.commit()
    return wallet
\end{lstlisting}

\textbf{Önemli:} Aylık otomatik grant sistemi kaldırılmıştır. Yorum satırında belirtildiği üzere, yenileme yalnızca paket satın alma döngüsünde gerçekleşir.

\subsubsection{Kota Kontrol Mekanizması}
Her AI isteği öncesinde \texttt{check\_quota\_available()} fonksiyonu çağrılır. Kontrol sırası:

\begin{enumerate}
  \item \textbf{Günlük limit} --- \texttt{daily\_used + tokens\_needed > daily\_limit} ise istek reddedilir.
  \item \textbf{Haftalık limit} --- \texttt{weekly\_used + tokens\_needed > weekly\_limit} ise istek reddedilir.
  \item \textbf{Bakiye kontrolü} --- \texttt{balance < tokens\_needed} ise istek reddedilir.
\end{enumerate}

\begin{lstlisting}[language=Python, caption={Kota kontrol fonksiyonu}]
def check_quota_available(wallet, tokens_needed=1):
    reset_quota_if_needed(wallet)
    if wallet.daily_used + tokens_needed > wallet.daily_limit:
        return False, "Gunluk limit asildi."
    if wallet.weekly_used + tokens_needed > wallet.weekly_limit:
        return False, "Haftalik limit asildi."
    if wallet.balance < tokens_needed:
        return False, "Yetersiz token bakiyesi."
    return True, "OK"
\end{lstlisting}

\subsubsection{Token Harcama ve Kota Güncelleme}
\begin{lstlisting}[language=Python, caption={Token düşme fonksiyonu}]
def deduct_tokens_and_update_quota(wallet, amount):
    reset_quota_if_needed(wallet)
    wallet.balance -= amount
    wallet.total_spent += amount
    wallet.weekly_used += amount
    wallet.daily_used += amount
    db.session.commit()
\end{lstlisting}

\subsection{Otomatik Yenileme (Auto-Renewal) Toggle}

\subsubsection{İş Kuralları}
\begin{itemize}
  \item Yalnızca en az bir paket satın almış kullanıcılar otomatik yenilemeyi aktif edebilir.
  \item Hediye (signup grant) tokenları otomatik yenilenemez.
  \item Yenileme günü, kullanıcının üyelik tarihine (kayıt olduğu gün) göre belirlenir (maks.\ 28).
  \item Mevcut sistem \textbf{Stripe üzerinden otomatik ödeme almaz}; yenileme, satın alınan token miktarını bakiyeye ekler.
\end{itemize}

\subsubsection{Backend Endpoint'leri}

\paragraph{GET /api/tokens/renewal-status}
Kullanıcının yenileme durumunu sorgular.

\begin{lstlisting}[language=Python, caption={Renewal status GET endpoint}]
@app.route('/api/tokens/renewal-status', methods=['GET'])
@jwt_required()
def get_renewal_status():
    user = get_current_user()
    wallet = get_or_create_token_balance(user)
    has_purchase = TokenPurchase.query.filter_by(
        user_id=user.id, status='completed'
    ).first() is not None
    return jsonify({
        'monthly_renewal_enabled': wallet.monthly_renewal_enabled,
        'monthly_renewal_day': wallet.monthly_renewal_day,
        'last_renewal_at': wallet.last_renewal_at,
        'can_enable': has_purchase
    })
\end{lstlisting}

\paragraph{PUT /api/tokens/renewal-status}
Kullanıcının yenileme durumunu günceller. Satın alma geçmişi yoksa 400 hatası döner.

\begin{lstlisting}[language=Python, caption={Renewal status PUT endpoint}]
@app.route('/api/tokens/renewal-status', methods=['PUT'])
@jwt_required()
def update_renewal_status():
    user = get_current_user()
    data = request.get_json(silent=True) or {}
    enabled = data.get('enabled')
    if enabled:
        has_purchase = TokenPurchase.query.filter_by(
            user_id=user.id, status='completed'
        ).first()
        if not has_purchase:
            return jsonify({
                'error': 'Paket satin almaniz gereklidir.'
            }), 400
    wallet = get_or_create_token_balance(user)
    wallet.monthly_renewal_enabled = bool(enabled)
    if wallet.monthly_renewal_enabled and not wallet.monthly_renewal_day:
        wallet.monthly_renewal_day = min(28, user.created_at.day if user.created_at else _utcnow().day)
    db.session.commit()
    return jsonify({
        'success': True,
        'monthly_renewal_enabled': wallet.monthly_renewal_enabled,
        'monthly_renewal_day': wallet.monthly_renewal_day
    })
\end{lstlisting}

\subsection{Aylık Yenileme Mekanizması}
\texttt{check\_and\_apply\_monthly\_renewal()} fonksiyonu, satın alma kaydındaki \texttt{auto\_renew=True} ve \texttt{renewal\_day} bilgisine göre çalışır:

\begin{lstlisting}[language=Python, caption={Aylık yenileme kontrolü}]
def check_and_apply_monthly_renewal(user_id):
    purchase = TokenPurchase.query.filter_by(
        user_id=user_id, status='completed', auto_renew=True
    ).order_by(TokenPurchase.completed_at.desc()).first()
    if not purchase or not purchase.renewal_day:
        return False
    # Sonraki yenileme tarihini hesapla
    # Tarih geldiyse token ekle ve logla
    if now >= next_renewal_date:
        wallet.balance += purchase.tokens_granted
        purchase.last_renewal_at = now
        # TokenTransaction kaydı olustur
        db.session.commit()
        return True
    return False
\end{lstlisting}

\subsection{Veritabanı Migration}
Mevcut Supabase (PostgreSQL) veritabanına eksik sütunlar \texttt{ALTER TABLE} komutlarıyla eklenmiştir. Migration, Docker konteynırı içinden Python scripti aracılığıyla çalıştırılmıştır:

\begin{lstlisting}[language=Python, caption={Migration scripti (Docker exec ile çalıştırıldı)}]
from sqlalchemy import text
cols = [
    ('token_balance', 'weekly_limit', 'INTEGER DEFAULT 1000'),
    ('token_balance', 'weekly_used', 'INTEGER DEFAULT 0'),
    # ... diger alanlar
    ('token_purchase', 'auto_renew', 'BOOLEAN DEFAULT FALSE'),
    ('token_purchase', 'renewal_day', 'INTEGER'),
    ('token_purchase', 'last_renewal_at', 'TIMESTAMP')
]
for table, col, col_type in cols:
    db.session.execute(
        text(f'ALTER TABLE {table} ADD COLUMN {col} {col_type}')
    )
    db.session.commit()
\end{lstlisting}

\section{E-posta Yapılandırması (Cloudflare Email Routing)}
% ============================================================

\subsection{Platform Seçimi}
Zoho Mail denemesi yapılmış, teknik kısıtlamalar nedeniyle \textbf{Cloudflare Email Routing} çözümüne geçilmiştir. Cloudflare hesabı mevcut Gmail adresi (\texttt{dilakemer1@gmail.com}) üzerinden aktif edilmiştir.

\subsection{DNS ve Nameserver Yönetimi}
Alan adının yönetim yetkisi İsimtescil üzerinden Cloudflare'e devredilmiştir:

\begin{itemize}
  \item \textbf{Eski Nameserver'lar:} \texttt{tr.dnsenable.com}, \texttt{eu.dnsenable.com} (silindi)
  \item \textbf{Yeni Nameserver'lar:}
  \begin{enumerate}
    \item \texttt{aitana.ns.cloudflare.com}
    \item \texttt{tom.ns.cloudflare.com}
  \end{enumerate}
\end{itemize}

\subsection{E-posta Yönlendirme Kurulumu}
\begin{itemize}
  \item \textbf{Özel Adres:} \texttt{info@codealchemist.com.tr}
  \item \textbf{Hedef Adres:} Gelen tüm e-postalar \texttt{dilakemer1@gmail.com} adresine yönlendirilmektedir.
  \item \textbf{Doğrulama:} Hedef e-posta adresine gönderilen onay bağlantısı ile tamamlanmıştır.
\end{itemize}

\subsection{Güvenlik ve Teslimat DNS Kayıtları}
\begin{itemize}
  \item \textbf{MX Kayıtları:} 3 adet MX kaydı ile e-posta trafiği Cloudflare'e yönlendirilmiştir.
  \item \textbf{SPF Kaydı:} \texttt{v=spf1 include:\_spf.mx.cloudflare.net \textasciitilde all}
  \item \textbf{DKIM Kaydı:} Dijital imza doğrulama kaydı eklenmiştir.
\end{itemize}

\subsection{Landing Page Entegrasyonu}
\texttt{LandingPage.jsx} footer bölümüne ``BİZE ULAŞIN'' başlığı ve \texttt{info@codealchemist.com.tr} e-posta bağlantısı eklenmiştir.

\section{Google OAuth Entegrasyonu}
% ============================================================

\subsection{Yapılandırma}
Google Cloud Console üzerinden OAuth 2.0 istemci kimliği oluşturulmuştur.

\begin{itemize}
  \item \textbf{Client ID:} \texttt{.env} ve \texttt{client/.env.local} dosyalarında \texttt{GOOGLE\_CLIENT\_ID} olarak tanımlıdır.
  \item \textbf{Frontend:} \texttt{VITE\_GOOGLE\_CLIENT\_ID} ortam değişkeni ile Vite build sürecine enjekte edilir.
\end{itemize}

\subsection{Backend Endpoint}
\texttt{POST /api/auth/google} endpoint'i, Google'dan alınan \texttt{credential} (JWT token) değerini doğrular ve kullanıcıyı oluşturur ya da giriş yapar.

\texttt{AuthModal.jsx} dosyasında Google Identity Services (GIS) kütüphanesi kullanılarak ``Continue with Google'' butonu render edilmektedir. Buton yalnızca login modunda ve \texttt{GOOGLE\_CLIENT\_ID} tanımlıysa gösterilir.
