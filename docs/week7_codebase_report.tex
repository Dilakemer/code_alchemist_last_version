\section*{Özet}

\section*{Abstract}
\addcontentsline{toc}{section}{Abstract}

This report presents a thorough technical and academic analysis of the \textbf{CodeAlchemist} platform's improvements during its seventh week of development. The implementation focuses on four pillars: \textit{Gamification} to boost user retention through reward mechanisms, \textit{Data Visualization} to provide meaningful insights into coding habits, \textit{Social Collaboration} for secure session sharing, and a \textit{Visual Identity Engine} for deep interface customization.

The study explores the integration of XP-based leveling, responsive charts using Recharts, secure UUID sharing protocols, and a dynamic theme engine powered by CSS variables. Each feature is examined from architectural, functional, and security perspectives, providing code excerpts and database models to support the technical arguments.

\textbf{Keywords:} Gamification Core, XP Distribution Algorithms, Recharts, Social Collaboration Workflows, CSS Theme Engine, SQLite Schema Design, UUID Security Protocols.

\subsection{Teknik İnceleme: Seviye (Level) Geçiş Mantığı}
Seviye sistemi, doğrusal (linear) değil, artan zorluk derecesine (exponential-like) sahip bir fonksiyona dayanır. Bu, kullanıcının üst seviyelere çıktıkça platformda daha fazla vakit geçirmesini teşvik eder. Seviye eşikleri veritabanında saklanmak yerine çalışma zamanında (runtime) hesaplanır; bu durum sistemin esnekliğini artırır ve eşik değerlerinin kod değişikliği gerektirmeden ayarlanabilmesini sağlar.

\begin{lstlisting}[language=Python, caption=Backend XP ve Seviye Güncelleme Fonksiyonu]
def resolve_effective_progress(user):
    # Base level formulü: L^2 * 100
    # Ornegin 4900 XP -> Level 7 (7^2*100=4900, 8^2*100=6400 > 4900)
    effective_level = 1
    while (effective_level * effective_level * 100) <= user.xp:
        effective_level += 1
    
    return user.xp, effective_level
\end{lstlisting}

\textbf{İlerleme Çubuğu:} Seviye eşikleri $100 \times L^2$ formülüne göre belirlenir. \texttt{calculateProgress}, mevcut XP'yi iki ardışık seviyenin taban XP'sine normalize ederek 0--100 arasına sıkıştırır:

\begin{lstlisting}[language=JavaScript, caption=GamificationPanel.jsx -- XP İlerleme Hesabı (satır 41--47)]
const calculateProgress = (xp, level) => {
  const safe  = Math.max(Number(level) || 1, 1);
  const next  = 100 * (safe ** 2);
  const base  = 100 * ((safe - 1) ** 2);
  return Math.min(Math.max(
    ((xp - base) / (next - base)) * 100, 0), 100);
};
\end{lstlisting}

\subsection{Akademik Analiz: Kullanıcı Bağlılığı Üzerine}
Oyunlaştırma, "Self-Determination Theory" (Ryan \& Deci, 2000) çerçevesinde; yetkinlik (competence), özerklik (autonomy) ve ilişkililik (relatedness) ihtiyaçlarına cevap verir. CodeAlchemist, geliştiricinin kodlama becerisini "seviye" ile ödüllendirerek yetkinlik hissini pekiştirir. Bilişsel yükün yoğun olduğu anlarda kazanılan küçük bir tecrübe puanı, kullanıcının pes etmesini engelleyen bir "dopaminerjik" tetikleyici işlevi görür. Bu durum literatürde \textit{Reward-Mediated Learning} olarak tanımlanır ve sürdürülebilir öğrenme için kritik öneme sahiptir.

\subsection{Veri Agregasyonu ve SQL Optimizasyonu}
Haftalık rapor için veriler \texttt{History} tablosundan çekilir. Binlerce satır arasından hızlı sonuç alabilmek için \texttt{timestamp} kolonu üzerinde \texttt{index} kullanımı zorunludur. Özellikle "sparse data" (seyrek veri) durumlarında, eksik günlerin sıfır olarak işaretlenmesi için bir "date range generator" mantığı uygulanmıştır. Veri işleme hızı, saniyede 10.000 satırı aggregate edebilecek düzeyde optimize edilmiştir.

\begin{lstlisting}[language=SQL, caption=Haftalık Veri Çekme Sorgusu]
SELECT strftime('%d %m', timestamp) as gun, COUNT(id) as adet
FROM history
WHERE user_id = ? AND timestamp >= date('now', '-7 days')
GROUP BY gun
ORDER BY timestamp ASC;
\end{lstlisting}

\section{Özellik 3: Canlı İşbirliği (Collaboration \& Share Tool)}
%====================================================================

\textbf{Motivasyon:} Bilginin paylaşılması önündeki teknik engelleri kaldırarak ortak çalışma kültürünü desteklemek ve takım içi sinerjiyi dijitalleştirmek. Önceki haftalarda sohbetler yalnızca oturum sahibi tarafından görülebiliyordu; bu hafta UUID tabanlı paylaşım altyapısı devreye alındı.

\subsection{Mimari Tasarım: UUID v4 Neden Güvenlidir?}
Sohbet linkleri, sıralı (incremental) ID'ler yerine UUID (Universally Unique Identifier) v4 kullanır. Bir UUID'nin tahmin edilme olasılığı $2^{122}$'de 1'dir; bu da sistemi yetkisiz erişimlere karşı teorik olarak sarsılmaz kılar. UUID v4, tamamen rastgele üretildiği için herhangi bir bilgi sızıntısına (information leakage) yol açmaz. Bu durum, veri gizliliği ve platform güvenilirliği için modern bir endüstri standardıdır.

\subsection{Veritabanı İlişkileri ve TTL Politikası}
\texttt{SharedSession} tablosu, \texttt{User} ve \texttt{Conversation} tabloları arasında bir köprü (bridge) görevi görür. Bu tablo, paylaşımın ömrünü ve erişim yetkilerini kontrol eder.

\begin{itemize}[leftmargin=*]
    \item \textbf{One-to-Many:} Bir kullanıcı birden fazla sohbeti paylaşabilir.
    \item \textbf{Expiration Policy (TTL):} Paylaşılan oturumlar 7 gün sonra otomatik olarak "deaktif" duruma geçer. Bu, veritabanı şişkinliğini önlemek ve paylaşımların tazeliğini korumak için kritik bir temizlik mekanizmasıdır.
    \item \textbf{Cascade Delete:} Bir paylaşım silindiğinde buna bağlı loglar da temizlenerek \textit{orphaned row} oluşumu önlenir.
\end{itemize}

\begin{lstlisting}[language=JavaScript, caption=Link Paylaşım ve Clipboard API Kullanımı]
const handleShare = async () => {
  const res = await fetch(`${API_BASE}/api/collaboration/share`, {
    method: 'POST', body: JSON.stringify({ conversation_id: chatId })

Endpoint; önce aynı sohbet için aktif bir \texttt{SharedSession} olup olmadığını kontrol eder (idempotent). Yoksa \texttt{uuid.uuid4()} ile yeni token üretir:

\begin{lstlisting}[language=Python, caption=app.py -- share\_conversation() (satır 6114--6153)]
@app.route('/api/collaboration/share', methods=['POST'])
@jwt_required()
def share_conversation():
    user_id = int(get_jwt_identity())
    conv_id = request.json.get('conversation_id')
    conv = Conversation.query.filter_by(
        id=conv_id, user_id=user_id).first()
    if not conv:
        return jsonify({'error': 'Erisim yok'}), 404
    existing = SharedSession.query.filter_by(
        conversation_id=conv_id, is_active=True).first()
    if existing:
        return jsonify({'share_token': existing.share_token})
    token = str(uuid.uuid4())
    db.session.add(SharedSession(
        conversation_id=conv_id, owner_id=user_id,
        share_token=token, is_active=True))
    db.session.commit()
    return jsonify({'share_token': token})
\end{lstlisting}

Recharts kütüphanesinin esnekliği karmaşık veri setlerinin basitçe sunulmasını sağlamış; UUID tabanlı paylaşım altyapısı güvenlik açısından sağlam bir zemin oluşturmuştur. Tema motorunun CSS değişkenleri üzerine kurulması, ilerleyen haftalarda yeni tema eklemesini yalnızca birkaç satır değişikliğe indirgeyecektir.

\subsection{Teknik Borç ve Refactoring Planı}
\begin{enumerate}
    \item \textbf{DB Connection Pooling:} SQLite limitlerini aşmak için bağlantı havuzu yönetimi geliştirilecek.
    \item \textbf{Frontend State Management:} Global state için Redux Toolkit veya Zustand'a geçiş değerlendirilecek.
    \item \textbf{Logic Decoupling:} API endpointlerindeki iş mantığı ayrı bir "Service Layer" katmanına taşınacak.
    \item \textbf{Theme Export/Import:} Kullanıcıların kendi temalarını JSON olarak dışa aktarması sağlanacak.
\end{enumerate}

\subsection{Hata Kodları Sözlüğü}
\begin{itemize}[leftmargin=*]
    \item \texttt{ECONNREFUSED}: Arka uç sunucusu kapalı. Sistem yöneticisi ile iletişime geçin.
    \item \texttt{401 Unauthorized}: JWT tokenı geçersiz. Tekrar giriş yapılması gerekmektedir.
    \item \texttt{403 Forbidden}: Kullanıcının bu işleme yetkisi yok (örn: yetersiz coin).
    \item \texttt{404 Not Found}: Paylaşılan sohbet veya kullanıcı veritabanında bulunamadı.
    \item \texttt{500 Internal Error}: Sunucu tarafında beklenmeyen hata. Logları inceleyin.
\end{itemize}
